## 1. Цель работы

Целью работы является построение и сравнение двух моделей нейронных сетей для аппроксимации сложной нелинейной функции двух переменных:

$$f(x,y) = \sin(x+2y) \cdot \exp\!\left(-(2x+y)^2\right), \quad x,y \in [-10,10].$$

Базовая модель обучается с фиксированной архитектурой, а вторая модель – с гиперпараметрами, подобранными автоматически с помощью библиотеки **Optuna**. Задача – минимизировать среднеквадратичную ошибку (MSE) на тестовой выборке.

## 2. Данные

- **Генерация** — 20 000 точек, равномерно распределённых в квадрате $[-10,10]^2$.
- **Разделение** — 70% (14 000) на обучение, 15% (3 000) на валидацию, 15% (3 000) на тест. Фиксированный случайный seed (`42`) обеспечивает воспроизводимость.
- **Особенности** — целевая функция имеет сложный рельеф: несколько локальных экстремумов и быстрое затухание при удалении от начала координат.

## 3. Методология

### 3.1. Базовая модель

**Архитектура** — полносвязная нейронная сеть:

- Входной слой: 2 нейрона.
- Два скрытых слоя по 64 нейрона, функция активации **ReLU**.
- Выходной слой: 1 нейрон (линейный).

**Параметры обучения:**

| Параметр | Значение |
| :--- | :--- |
| Оптимизатор | **Adam** (`lr = 0.001`) |
| Функция потерь | MSE |
| Эпохи | 200 |
| Размер батча | 64 |

### 3.2. Оптимизация гиперпараметров (Optuna)

Для поиска лучшей конфигурации проведено **10 испытаний**. Варьировались следующие параметры:

| Параметр | Диапазон / варианты |
| :--- | :--- |
| Число скрытых слоёв | 1 – 4 |
| Размер скрытого слоя | 32, 64, 128, 256 |
| Функция активации | ReLU, Tanh, LeakyReLU |
| Скорость обучения | логарифмически равномерно от $10^{-4}$ до $10^{-2}$ |
| Размер батча | 32, 64, 128 |
| Оптимизатор | Adam, SGD (`momentum=0.9`), RMSprop |
| Функция потерь | MSE, HuberLoss, SmoothL1Loss |

Каждое испытание обучалось 50 эпох, цель — минимизировать MSE на валидационной выборке.

**Лучшие найденные гиперпараметры:**

```python
{
    'n_layers': 4,
    'hidden_units': 128,
    'activation': 'Tanh',
    'lr': 0.0006775548129944136,
    'batch_size': 64,
    'optimizer': 'Adam',
    'loss': 'Huber'
}
```

### 3.3. Финальная модель

- Обучалась на объединённых обучающей и валидационной выборках (17 000 точек).
- Использованы лучшие гиперпараметры из Optuna.
- Длительность обучения — 200 эпох.
- Контрольная метрика — MSE на тестовой выборке.

## 4. Результаты

### 4.1. Количественное сравнение

| Модель | Test MSE |
| :--- | :--- |
| Базовая (ReLU, 2×64) | 0.003054 |
| Оптимизированная (Optuna) | 0.000129 |

**Относительное улучшение:** MSE снизилась более чем в **23 раза** (с $3.05 \times 10^{-3}$ до $1.29 \times 10^{-4}$).

- **Истинная функция** демонстрирует сложную картину: два ярко выраженных пика (положительный и отрицательный) в центральной области и быстрое затухание к краям.

- **Optuna-модель** практически неотличима от истинной функции по визуальному качеству — контуры совпадают, включая мелкие детали.

## 5. Выводы

1. **Существенное влияние гиперпараметров** — подбор архитектуры (глубина 4, ширина 128, активация Tanh), скорости обучения, оптимизатора и функции потерь позволил драматически улучшить точность модели.
2. **Функция потерь Huber** в сочетании с Adam показала лучший результат, что говорит о её устойчивости к возможным выбросам в данных.
3. **Визуально оптимизированная модель** практически идеально воспроизводит истинную функцию, в то время как базовая модель страдает от недообучения (ограниченная ёмкость сети и неоптимальные параметры).
4. **Рекомендации** — при решении задач регрессии со сложной нелинейной зависимостью целесообразно автоматизировать подбор гиперпараметров (например, с помощью Optuna). Это позволяет значительно повысить качество модели без ручного перебора.
